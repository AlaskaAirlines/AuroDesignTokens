## Feature: CSS Theme Selectors Split into Color and Typography — `@aurodesignsystem/design-tokens` v9.2.0

### Summary

Version 9.2.0 changes how CSS custom properties are grouped under `data-aag-theme` selectors. Previously, all tokens for a theme were placed under a single `[data-aag-theme="aag-theme-<code>"]` selector. They are now split into two selector groups — **color** and **typography** — each with an additional, more specific selector variant. This is a non-breaking change; existing markup continues to work without modification.

---

### Why it is not a breaking change

The base `[data-aag-theme="aag-theme-<code>"]` selector is still present on every CSS block. Any existing markup using `data-aag-theme` will continue to receive all tokens — both color and typography — without modification. The new `-color` and `-type` selector variants are purely additive.

---

### What changed

**Before (v9.1.x):**
```css
[data-aag-theme="aag-theme-as"] {
  --ds-advanced-color-button-primary-background: #01426a;
  --ds-basic-type-family-body: "AS Circular";
  /* ... all tokens together ... */
}
```

**After (v9.2.0):**
```css
[data-aag-theme="aag-theme-as"], [data-aag-theme="aag-theme-as-color"] {
  --ds-advanced-color-button-primary-background: #01426a;
  /* ... color tokens only ... */
}

[data-aag-theme="aag-theme-as"], [data-aag-theme="aag-theme-as-type"] {
  --ds-basic-type-family-body: "AS Circular";
  /* ... typography tokens only ... */
}
```

Token classification is based on the CSS custom property name:
- **Color tokens**: properties containing `-color-` (e.g. `--ds-advanced-color-*`, `--ds-basic-color-*`)
- **Typography tokens**: properties containing `-type-` (e.g. `--ds-basic-type-*`)
- **Other tokens**: any remaining tokens are placed under the base `[data-aag-theme="aag-theme-<code>"]` selector only

### Affected files

| File | Change |
|---|---|
| `dist/themes/CSSCustomProperties--bundled.css` | Each theme block split into color and typography selector groups |
| `dist/themes/*/CSSCustomProperties--*.min.css` | Minified versions regenerated |
| `dist/web/alaska.css` | Own-theme and other-theme blocks split by color/typography |
| `dist/web/atmos.css` | Same |
| `dist/web/hawaiian.css` | Same |
| `dist/web/*.min.css` | Minified versions regenerated |
| `scripts/transformCSS.mjs` | Build logic updated to classify and split tokens |

---

### Consumer impact

**Low risk for most consumers.** The base `[data-aag-theme="aag-theme-<code>"]` selector still appears on every block, so existing markup using `data-aag-theme` will continue to receive all tokens without changes.

However:

- **If you rely on all tokens appearing in a single CSS rule block** (e.g. parsing the CSS output programmatically), your code will now see multiple blocks per theme instead of one.
- **If you minify or post-process the distributed CSS** with tooling that deduplicates selectors, verify that the repeated `[data-aag-theme="..."]` selectors across the color and typography blocks are handled correctly.
- **The `dist/web/*.css` files now contain more selector blocks** per theme (2 per theme instead of 1). File sizes increase marginally.

---

### New capability for consumers

You can now independently override color or typography by using the new selector values:

```html
<!-- Apply Alaska color tokens only (keep current typography) -->
<div data-aag-theme="aag-theme-as-color">...</div>

<!-- Apply Alaska typography tokens only (keep current colors) -->
<div data-aag-theme="aag-theme-as-type">...</div>

<!-- Apply both (same as before) -->
<div data-aag-theme="aag-theme-as">...</div>
```

---

### Required action for consumers

1. **No action required** if you use `data-aag-theme="aag-theme-<code>"` — behavior is unchanged.
2. If you parse the distributed CSS files programmatically, update your parsing logic to handle multiple selector blocks per theme.
3. Optionally, adopt `aag-theme-<code>-color` or `aag-theme-<code>-type` attribute values to independently control color and typography theming.
